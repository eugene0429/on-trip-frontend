# Page Override: Trip Profile (S-05 일행 프로필)

## 레이아웃

```

│ ←                       ⋯    │  ← back / 더보기(신고)

│   ┌────────────────────┐   │

│   │                    │   │     크기 화면 너비 - 32dp, 정사각형
│   │  @닉네임 (display) │   │
```

## 핵심 사양

- **사진 영역:** 1:1 정사각형, max 320dp. 회전 -1deg로 살짝 기울임 (강조 카드).
- **사진 없음:** 노랑 그라데이션(없음 — 단색) + 큰 닉네임 첫 글자 (Pretendard 800, 96px).
- **닉네임/MBTI/나이:** 사진 카드 하단 inset 또는 카드 아래 별도 섹션. PRD에 따라 PASS 인증된 나이만 표시(닉네임은 사용자 입력).
- **위치 칩:** "📍 광안리 등록 중" (`accentLime` 배경 + outline + 점 표시).
- **일행 정보 카드:** 4행 정렬. 라벨(textMuted, caption) + 값(text, body). 컨셉 행은 Chip 가로 스크롤.
- **CTA fixed bottom:** safe-area-inset-bottom + 16dp. 스크롤 콘텐츠와 분리.

## 인터랙션

- **사진 탭:** 풀스크린 lightbox (StickerCard 안에 줌 가능 이미지). swipe-down 닫기.
- **콕 CTA 탭:** mechanical press + impactMedium → 콕 모달(poke-modal) 열림.
- **신고 더보기:** 화면 우상단 ⋯ 탭 → 액션시트 (신고 / 차단 / 공유 — 공유는 V2).
- **스크롤:** 사진 영역까지 paragllaxHeader 효과 — V2. MVP는 단순 스크롤.

## 상태

- **이미 콕 보낸 일행:** CTA가 "콕 보냈어요 · 응답 대기" (disabled secondary) + 회전 펄스 1회 (300ms).
- **이미 매칭됨:** CTA가 "채팅하기" (primary, lime 배경) → 채팅방 navigate.
- **상대방이 차단함:** 화면 진입 자체 차단 → "이 일행을 볼 수 없어요" alert 후 뒤로.
- **로딩:** 사진 영역 skeleton + 정보 카드 회색 라인.

## 콕 보낸 후 우상단 노랑 배지

이 일행을 이미 콕 보낸 상태면 사진 카드 우상단에 작은 회전된 스티커 배지: "콕 보냄 ✓" (FFE066, outline, rotate -3deg).

## 안티패턴

- ❌ 사진 클릭으로 댓글/리액션 (데이팅 앱 색채)
- ❌ "팔로우" 같은 SNS 액션
- ❌ 한줄소개를 펼치는 별도 모달 (그냥 풀텍스트 노출)
- ❌ CTA 위치를 화면 중앙/상단 (반드시 fixed bottom)
