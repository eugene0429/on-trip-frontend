# Page Override: Map (S-03 홈)

> 마스터 디자인 시스템 위에 이 화면 전용 규칙. 마스터와 충돌 시 본 문서 우선.

## 레이아웃

```
┌─────────────────────────────┐
│ [상단 SafeArea]             │
│ ┌───┐ On-Trip      ┌─┐ ┌─┐ │  ← 좌: 로고 칩
│ └───┘             [활][마] │     우: Activity(뱃지)·My
├─────────────────────────────┤
│                             │
│       카카오맵 영역         │
│   (전체 화면 fill)          │
│                             │
│   📍 광역 줌: 시·도 인구 핀 │

│                             │
│              ┌─────┐        │
│              │ ⊕ │        │  ← 내 위치 버튼 (우하단, hitSlop)

│                             │
├─────────────────────────────┤
│  ┌─────────────────────┐    │

│  [하단 Tab bar]             │
```

## 핵심 사양

- **지도:** 카카오맵 SDK (RN). 초기 줌은 사용자 GPS 권한 허용 시 현재 위치 중심, 거부 시 서울 중심 + 권한 요청 모달.
- **줌 임계:** 줌 레벨 ≤ 8 → 광역(행정구역), > 8 → 세부(POI). PinMarker 자동 전환.
- **인구 새로고침:** 60초 캐시(서버 측). 사용자가 지도를 50% 이상 이동하면 강제 재요청.
- **하단 CTA:** safe-area-inset-bottom + 16dp. 스크롤 X (지도는 제스처).

## PinMarker (지도 핀) 색상 단계

| 인원수 | Background | Text |
|---|---|---|
| 1–9 | `surface` (#FFF) | outline |
| 10–29 | `primary` (#FFE066) | outline |
| 30–69 | `accentPink` (#FFB3D9) | outline |
| 70+ | `accentRed` (#FF5A5A) | white |

핀 크기 = `32 + min(인구/3, 32)` dp (최대 64dp).

## 인터랙션

- **핀 탭:** 햅틱 `selection` + 1.1배 스케일 스프링(120ms) → 내려올 때 거점 상세로 navigate.
- **CTA "여행 왔어요":** mechanical press + impactMedium → 등록 화면 이동.
- **내 위치 버튼:** GPS 재요청 + 지도 카메라 애니메이션 (400ms ease-out).

## 빈/오류 상태

- **GPS 권한 거부:** 서울 중심 + 화면 하단에 노랑 토스트 카드 "GPS를 켜면 근처 일행을 더 잘 볼 수 있어요" + [설정 열기] 버튼.
- **인구 0:** "이 지역엔 아직 아무도 없어요. 먼저 등록해 볼까요?" 토스트.
- **지도 SDK 로드 실패:** 전체 화면 placeholder + 재시도 버튼.

## 안티패턴 (이 화면 한정)

- ❌ 핀 위에 그림자 적용 (지도 위 시각 노이즈)
- ❌ 핀에 회전 적용 (가독성 저하)
- ❌ CTA를 지도 위 floating 으로 (safe-area + 하단 고정 유지)
- ❌ 줌 레벨 무관한 균일 핀 크기
