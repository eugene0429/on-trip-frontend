# Page Override: Activity (S-08, S-09 받은 콕 / 보낸 콕 / 매칭)

## 레이아웃

```
┌─────────────────────────────┐
│ [SafeArea]                  │
│ 활동                         │  ← 좌상단 display
├─────────────────────────────┤
│  ┌─[받은 콕]─[보낸 콕]─[매칭]┐│  ← 세그먼트 컨트롤 (StickerCard 기반)
│  │ ●3       1      2       │  ← 활성 탭에 노랑 점 + 카운트 배지
│  └──────────────────────────┘│
├─────────────────────────────┤
│                             │
│  ┌─────────────────────┐    │  ← 받은 콕: 정체 가려진 카드
│  │ ❓ 누군가가 콕 보냄│    │
│  │ 일행: 남2 여1 ·20후│    │
│  │ #맛집 #술자리      │    │

│  │ 2시간 전           │    │

│  │ │ 프로필 보기   │  │    │  ← StickerButton primary

│  └─────────────────────┘    │
│  [카드 2]                  │
└─────────────────────────────┘
```

## 세그먼트 컨트롤 (탭)

- 단일 stickerCard 컨테이너 (offset 4, outline 2.5) 안에 3개 세그먼트.
- 활성 세그먼트: 노랑 배경 + outline 보더 인셋 (살짝 들어간 효과).
- 비활성: 흰 배경.
- 우상단 카운트: 라임 점 + JetBrains Mono.

## 받은 콕 카드 (가려진 상태)

```
[원형 ❓ 일러스트 64dp · outline]   ← 정체 가린 placeholder
일행 정보 (정체 단서 X):
  - 인원, 성별 구성, 나이대, 컨셉, 지역
한마디 미리보기: 첫 10자 + "..."
시각: "n시간 전" / "어제" / "n일 전"
[프로필 보기] CTA — 500🪙 코인배지
```

**FR-034 부합:** 발신자 닉네임/사진/한줄소개는 절대 노출 X. 일행 정보 + 한마디 일부만.

**한마디 미리보기 길이:** 첫 10자 + "..." (Pretendard 500). 없으면 "(메시지 없음)" 회색.

## 받은 콕 → 프로필 열람 모달

[프로필 보기] 탭 시 confirmation 모달 (스티커 시트):

```
"이 콕의 프로필을 열어볼까요?"
"열람 시 500🪙 차감돼요. 이미 본 프로필은 무료예요."
[지금 열람]  [나중에]
```

확정 시 ProfileView 기록 + 일행 프로필 화면으로 navigate (정체 공개됨).

## 보낸 콕 카드

```
[작은 사진 48dp] @닉네임
일행: 남2 여1
"내가 보낸 한마디" 풀텍스트
3시간 전 · 응답 대기 / 매칭됨 / 만료
[액션 없음 또는 "다시 콕은 안 돼요" 회색]
```

상태 칩으로 right top:
- 응답 대기 (textMuted 칩)
- 매칭됨! (라임 배경 칩) — 탭하면 채팅방
- 만료 (회색 점선 outline 칩)

## 매칭 카드

```
[양쪽 사진 겹침 64dp + outline] 양쪽 닉네임
"💛 매칭 성사일: 5월 1일"  ← 매칭 시각
마지막 메시지 미리보기 (있으면)
[채팅하기] CTA primary
```

매칭은 별도 화면에서 강조 (매칭 성사 직후 자동 모달 — 별도 흐름).

## 인터랙션

- **세그먼트 전환:** 탭 시 selection 햅틱 + 콘텐츠 슬라이드 (옆으로 16dp 슬라이드 + 페이드).
- **카드 탭:** 받은 콕은 → 열람 confirmation 모달. 보낸 콕은 → 일행 프로필(이미 매칭 시 채팅방). 매칭은 → 채팅방.
- **풀투리프레시:** 모든 탭 지원. RefreshControl tintColor outline.

## 빈 상태

- **받은 콕 없음:** 큰 콕 일러스트 + "아직 콕 받지 않았어요" + "지도에서 다른 일행을 찾아보세요" + [지도로] CTA.
- **보낸 콕 없음:** "아직 누구에게도 콕을 보내지 않았어요" + [지도로] CTA.
- **매칭 없음:** "첫 매칭을 만들어 봐요!" + [지도로] CTA.

## 알림 뱃지

- 받은 콕 새로 들어옴 → 탭바 Activity에 빨강 점 뱃지 (count는 안 보여줌, 점만).
- 매칭 성사 시 → 풀스크린 축하 모달 (별도 페이지 — `match-success.md` V2)이 우선 표시되고, 그 후 활동 탭 매칭 칸에 추가됨.

## 안티패턴

- ❌ 받은 콕에서 발신자 닉네임/사진 노출 (BM 깨짐)
- ❌ 한마디 풀텍스트 노출 (10자 미리보기 — BM 동기)
- ❌ 매칭/일반 콕 시각 톤 동일 (매칭은 라임 배경으로 강조 차별화)
- ❌ 세그먼트 활성 탭 흐릿한 표시 (반드시 노랑 배경)
