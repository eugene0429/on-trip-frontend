# Page Override: Poke Modal (S-06 콕 보내기)

> 하단 슬라이드 업 시트. 일행 프로필에서 "콕 찌르기" 탭 시 진입.

## 레이아웃

```
┌─────────────────────────────┐
│  [스크림 50% 검정]           │
│                             │
│  ┌───────────────────────┐  │
│  │ ━━━ (시트 핸들)      │  │  ← 스와이프 다운 닫기
│  │ ┌─[사진]─┐ @닉네임   │  │
│  │ │       │ 광안리 일행 │  │
│  │ └───────┘            │  │
│  │ ─────────────────    │  │
│  │ "한마디 (선택)"      │  │  ← caption 라벨
│  │ ┌─────────────────┐ │  │
│  │ │ 50자 이내로...  │ │  │  ← StickerInput, 50자 카운터
│  │ │                 │ │  │     예시: "맛집 같이 도장깨기 해요!"
│  │ └─────────────────┘ │  │
│  │ ─────────────────    │  │
│  │  내 코인 850🪙       │  │  ← CoinBadge + 잔액 (caption)
│  │  ┌────────────────┐ │  │
│  │  │ 콕 보내기 100🪙│ │  │  ← StickerButton primary (offset 8)
│  │  └────────────────┘ │  │     amount 강조
│  │  [취소]              │  │  ← ghost
│  └───────────────────────┘  │
└─────────────────────────────┘
```

## 핵심 사양

- **시트 컴포넌트:** `@gorhom/bottom-sheet` snap points: ['CONTENT_HEIGHT'].
- **시트 background:** surface 흰색 + outline 2.5 (위쪽만) + shadow.lg.
- **시트 코너:** 상단만 radii.xl(28). 하단은 safe-area-inset-bottom 처리.
- **핸들:** 36×4 회색 둥근 막대.
- **닫기 방식 3종:** 핸들 스와이프 / 스크림 탭 / [취소] 버튼.

## 한마디 입력 (StickerInput)

- multiline 2줄, 폰트 body, placeholder caption color.
- 50자 카운터: 입력 우하단 작은 caption (`30/50` 형태).
- 자동 마스킹: 클라이언트는 표시만, 서버 처리. 입력 시 즉시 마스킹은 V2 (UX 복잡).
- IME 한글 입력 시 placeholder 자동 사라짐 ✓ (RN 기본 동작).
- 자동 포커스 X (사용자가 의도해서 탭하게).

## 코인 차감 표기

```
[코인 ICON ◯ 노랑] 콕 보내기 100🪙
```

100을 JetBrains Mono 700, 노랑 배경 + outline 칩(작은 그림자). 사용자에게 차감 비용을 명확히.

## 인터랙션

- **모달 진입:** 250ms ease-out spring, 스크림 페이드 인.
- **모달 퇴장:** 180ms ease-in.
- **콕 버튼 탭:** mechanical press + 즉시 비활성 (중복 발사 차단) + impactMedium 햅틱.
- **API 응답 후:**
  - 성공: 시트 닫힘 + 화면 상단 토스트 (라임 배경) "콕 보냈어요 ✨ · 100🪙 사용". 토스트 5초.
  - 실패 (코인 부족): 시트 유지 + 빨간 토스트 "코인이 부족해요" + [충전] 버튼 (V2 활성, MVP는 비활성 + "곧 오픈").
  - 실패 (이미 콕 보냄): 시트 닫힘 + 회색 토스트 "이미 콕을 보낸 일행이에요".
- **스와이프 다운:** 30% 이상 끌면 닫힘. 미만이면 스프링 복귀.

## 상태

- **코인 부족 (잔액 < 100):** [콕 보내기] 버튼 disabled + 부족분 강조 ("100🪙 필요 · 잔액 50🪙").
- **메시지 비어있음:** 콕 보낼 수 있음 (한마디 선택 사항). 단, 받은 콕 미리보기에 "(메시지 없음)" 표기.
- **차단된 일행 또는 만료 일행:** 모달 자체 진입 차단 (이전 화면에서 이미 거름).

## 안티패턴

- ❌ 한마디 필수화 (선택으로 유지 — 진입 마찰 최소)
- ❌ 모달을 풀스크린으로 (시트 형태 유지)
- ❌ [콕 보내기] 버튼 코인 표기 누락 (반드시 amount 명시)
- ❌ 결제 유도 강조 (MVP는 "곧 오픈"으로 명확히)
- ❌ 한마디에 이모지 추천 칩 (MVP — 사용자 자유 입력만)
